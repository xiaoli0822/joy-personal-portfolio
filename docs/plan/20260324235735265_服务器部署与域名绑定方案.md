# 服务器部署与域名绑定方案

## 项目判断
- 技术栈：Next.js 14
- 运行方式：Node.js 常驻服务
- 反向代理：Nginx
- 持久化目录：
  - `content/site-content.json`
  - `public/uploads`
- 必备环境变量：
  - `ADMIN_PASSWORD`

## 推荐部署方案
推荐使用 Linux 云服务器 + Node.js 20 + Nginx + systemd + Certbot。

原因：
- 当前项目包含 `app/api/*` 接口，不能按纯静态站点部署。
- 后台会写入 `content/site-content.json` 和 `public/uploads`，需要服务器本地可写目录。
- 这套方案最稳，后续绑定域名、开 HTTPS、重启自恢复都简单。

## 服务器准备
以 Ubuntu 22.04/24.04 为例：

```bash
sudo apt update
sudo apt install -y nginx
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node -v
npm -v
```

建议创建目录并上传项目：

```bash
sudo mkdir -p /var/www/joy-personal-portfolio
sudo chown -R $USER:$USER /var/www/joy-personal-portfolio
```

## 项目部署
把仓库代码上传到服务器后，在项目目录执行：

```bash
npm install
cp .env.production.example .env.production
npm run build
```

编辑 `.env.production`：

```env
NODE_ENV=production
ADMIN_PASSWORD=改成你自己的强密码
```

## systemd 常驻运行
复制仓库内示例配置：

文件：`deploy/portfolio.service.example`

目标位置：

```bash
sudo cp deploy/portfolio.service.example /etc/systemd/system/portfolio.service
```

按实际情况修改：
- `User`
- `Group`
- `WorkingDirectory`
- `EnvironmentFile`

然后执行：

```bash
sudo systemctl daemon-reload
sudo systemctl enable portfolio
sudo systemctl start portfolio
sudo systemctl status portfolio
```

## Nginx 反向代理
复制仓库内示例配置：

文件：`deploy/nginx-portfolio.conf.example`

目标位置：

```bash
sudo cp deploy/nginx-portfolio.conf.example /etc/nginx/sites-available/joy-portfolio
```

把 `server_name` 改成你的域名，例如：

```nginx
server_name yourdomain.com www.yourdomain.com;
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/joy-portfolio /etc/nginx/sites-enabled/joy-portfolio
sudo nginx -t
sudo systemctl reload nginx
```

## 域名绑定
在你的域名 DNS 控制台配置：

- 根域名 `@`：
  - 记录类型：`A`
  - 记录值：你的服务器公网 IP
- `www`：
  - 记录类型：`A`
  - 记录值：你的服务器公网 IP

如果你只想保留一个主域名，可以后续在 Nginx 里做 `www` 到根域或根域到 `www` 的 301 跳转。

## HTTPS 证书
域名解析生效后执行：

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
```

完成后 Certbot 会自动修改 Nginx 配置并启用 HTTPS。

## 发布与更新
以后每次更新代码，进入项目目录执行：

```bash
git pull
npm install
npm run build
sudo systemctl restart portfolio
```

## 关键注意事项
- 后台密码不要继续使用当前仓库里的开发密码。
- `content/site-content.json` 和 `public/uploads` 是运行期会变更的数据，迁移或重装服务器前要备份。
- 如果开启了防火墙，记得放行 `80` 和 `443` 端口。
- 首次部署建议先只绑定一个域名，确认站点正常后再加 `www`。

- 域名访问返回站点首页
- HTTPS 证书签发成功
